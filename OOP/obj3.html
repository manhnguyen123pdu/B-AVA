13.1 Static là gì?

static là từ khóa trong Java chỉ định rằng một thành phần (biến, phương thức, khối) thuộc về lớp chứ không thuộc về đối tượng

Khái niệm cơ bản

Student.java - Ví dụ cơ bản 
class Student {
    // Instance variables (non-static)
    String name;    // Mỗi student có tên riêng
    int age;        // Mỗi student có tuổi riêng
    
    // Static variables
    static String schoolName = "Đại học Công nghệ"; // Tất cả student cùng trường
    static int totalStudents = 0; // Đếm tổng số student đã tạo
}

Đặc điểm chính của static:

  • Chỉ có một bản sao trong bộ nhớ, dù tạo bao nhiêu đối tượng
  • Được chia sẻ chung bởi tất cả các đối tượng của lớp
  • Tồn tại suốt chương trình (từ khi lớp được nạp đến khi kết thúc)
  • Truy cập trực tiếp thông qua tên lớp mà không cần tạo đối tượng

Kết quả:

1. Static Variable (Biến tĩnh)

1.1 Cú pháp

Cú pháp khai báo 
public class MyClass {
    static int staticVar;  // Biến tĩnh
    int instanceVar;        // Biến instance (thông thường)
}

1.2 Ví dụ minh họa

Counter.java - So sánh static vs non-static 
public class Counter {
    static int count = 0;  // Biến tĩnh
    int instanceCount = 0;  // Biến instance

    public Counter() {
        count++;          // Tăng biến tĩnh
        instanceCount++;  // Tăng biến instance
    }

    public static void main(String[] args) {
        Counter obj1 = new Counter();
        System.out.println("Static count: " + Counter.count);      // 1
        System.out.println("Instance count: " + obj1.instanceCount); // 1

        Counter obj2 = new Counter();
        System.out.println("Static count: " + Counter.count);      // 2 (tăng)
        System.out.println("Instance count: " + obj2.instanceCount); // 1 (vẫn là 1)
    }
}

Kết quả:

Static count: 1
Instance count: 1
Static count: 2
Instance count: 1

👉 Biến tĩnh count được chia sẻ giữa obj1 và obj2

1.3 Khi nào sử dụng biến tĩnh?

  • Lưu trữ giá trị chung (ví dụ: số lượng đối tượng được tạo)
  • Hằng số (kết hợp với final): static final double PI = 3.14159;
  • Chia sẻ dữ liệu giữa các đối tượng (ví dụ: biến đếm, cache, cấu hình)

1.4 Lưu ý quan trọng

  • Không thể truy cập trực tiếp biến tĩnh từ phương thức non-static (trừ khi thông qua đối tượng)
  • Tránh lạm dụng vì biến tĩnh tồn tại lâu trong bộ nhớ, có thể gây rò rỉ
  • Không an toàn luồng (non-thread-safe) trong môi trường đa luồng

1.5 So sánh biến tĩnh vs biến instance

Đặc điểm Biến tĩnh (static) Biến instance (non-static)
Số bản sao 1 cho cả lớp 1 cho mỗi đối tượng
Truy cập ClassName.variable object.variable
Bộ nhớ Cấp phát khi lớp được nạp Cấp phát khi tạo đối tượng
Phạm vi tồn tại Toàn bộ chương trình Tồn tại cùng đối tượng

1.6 Ví dụ thực tế

Employee.java - Ứng dụng thực tế 
public class Employee {
    static String company = "ABC Corp";  // Tên công ty chung
    String name;                         // Tên nhân viên riêng

    public Employee(String name) {
        this.name = name;
    }

    void display() {
        System.out.println(name + " works at " + company);
    }

    public static void main(String[] args) {
        Employee e1 = new Employee("Alice");
        Employee e2 = new Employee("Bob");
        
        e1.display();  // Alice works at ABC Corp
        e2.display();  // Bob works at ABC Corp
        
        // Thay đổi biến tĩnh ảnh hưởng đến tất cả
        Employee.company = "XYZ Corp";
        e1.display();  // Alice works at XYZ Corp
        e2.display();  // Bob works at XYZ Corp
    }
}

2. Static Method (Phương thức tĩnh)

Đặc điểm: Thuộc về lớp, không thuộc về đối tượng | Có thể gọi mà không cần tạo đối tượng: ClassName.methodName()

Quy tắc sử dụng static method:

  • Chỉ truy cập được biến static và gọi phương thức static khác
  • Không thể sử dụng từ khóa this
  • Không thể ghi đè (override)
MathUtils.java - Ví dụ về static method 
public class MathUtils {
    // Biến static
    private static int operationCount = 0;
    
    // Phương thức static
    public static int add(int a, int b) {
        operationCount++;
        return a + b;
    }
    
    public static int subtract(int a, int b) {
        operationCount++;
        return a - b;
    }
    
    public static int getOperationCount() {
        return operationCount;
    }
    
    // KHÔNG THỂ làm như sau:
    // public void instanceMethod() {
    //     // Không thể gọi từ static method
    // }
    
    // public static void useThis() {
    //     System.out.println(this);  // LỖI: không có 'this' trong static
    // }
}

// Sử dụng
int sum = MathUtils.add(5, 3);  // Không cần tạo object
int diff = MathUtils.subtract(10, 4);
System.out.println("Số phép tính: " + MathUtils.getOperationCount());

Factory Pattern với static method

Car.java - Factory methods 
public class Car {
    private String brand;
    private String model;
    
    // Constructor private
    private Car(String brand, String model) {
        this.brand = brand;
        this.model = model;
    }
    
    // Factory methods (static)
    public static Car createToyota(String model) {
        return new Car("Toyota", model);
    }
    
    public static Car createHonda(String model) {
        return new Car("Honda", model);
    }
    
    public static Car createBMW(String model) {
        return new Car("BMW", model);
    }
}

// Sử dụng
Car toyota = Car.createToyota("Camry");
Car honda = Car.createHonda("Civic");

3. Static Block (Khối tĩnh)

Mục đích: Thực thi code khi lớp được tải vào bộ nhớ | Khởi tạo biến static phức tạp | Load cấu hình, tài nguyên
DatabaseConfig.java - Ví dụ về static block 
import java.util.Properties;
import java.io.FileInputStream;
import java.io.IOException;

public class DatabaseConfig {
    // Biến static
    private static Properties config;
    
    // Static block - chạy khi class được tải
    static {
        System.out.println("Đang tải cấu hình database...");
        config = new Properties();
        
        try {
            // Load từ file
            config.load(new FileInputStream("database.properties"));
            System.out.println("Đã tải cấu hình thành công");
        } catch (IOException e) {
            System.out.println("Không tìm thấy file config, dùng mặc định");
            config.setProperty("url", "localhost:3306");
            config.setProperty("username", "root");
            config.setProperty("password", "123456");
        }
    }
    
    public static String getConfig(String key) {
        return config.getProperty(key);
    }
}

// Khi chạy chương trình:
// - Khi gọi DatabaseConfig.getConfig() lần đầu
// - Class DatabaseConfig được tải
// - Static block chạy
// - Sau đó mới thực thi getConfig()

Chú ý:

Static block được thực thi chỉ một lần khi class được tải vào bộ nhớ (JVM). Thứ tự thực thi:

  1. Tải class vào memory
  2. Khởi tạo static variables (về 0, null, false)
  3. Thực thi static blocks (theo thứ tự trong code)
  4. Các bước khác của class loading

4. Static Import (Import tĩnh)

Mục đích: Cho phép sử dụng trực tiếp các thành viên static mà không cần ghi tên class phía trước
Ví dụ so sánh có và không có static import 
// ====== KHÔNG dùng static import ======
double area = Math.PI * Math.pow(radius, 2);
System.out.println("Kết quả: " + result);

// ====== DÙNG static import ======
import static java.lang.Math.PI;
import static java.lang.Math.pow;
import static java.lang.System.out;

double area = PI * pow(radius, 2);
out.println("Kết quả: " + result);  // Ngắn gọn hơn

Lưu ý khi sử dụng static import:

  • Chỉ áp dụng cho: static variable, static method, static final (hằng số)
  • Nên dùng có chừng mực để tránh làm code khó hiểu
  • Không nên dùng với các method trùng tên từ nhiều class khác nhau

13.2 Khi nào dùng / không dùng static

Ưu điểm của static

  • Tiết kiệm bộ nhớ: Chỉ 1 bản sao cho cả lớp
  • Truy cập nhanh: Không cần tạo object
  • Chia sẻ dữ liệu: Toàn bộ chương trình có thể dùng chung

Nhược điểm của static

  • Khó test: Không thể mock trong unit test
  • Gây coupling: Liên kết chặt với class
  • Vấn đề đa luồng: Cần cẩn thận với thread safety

So sánh static vs non-static

Tiêu chí Static Non-Static
Thuộc về Lớp Đối tượng
Số bản sao 1 Mỗi object 1 bản
Tồn tại Khi class được tải Khi object được tạo
Truy cập ClassName.member object.member
Ghi đè Không thể Có thể
Từ khóa this Không có Có

Khi nào nên dùng static?

✅ NÊN DÙNG static trong các trường hợp sau: 
// 1. Hằng số
public static final double PI = 3.14;

// 2. Utility methods (phương thức tiện ích)
public static int add(int a, int b) { return a + b; }

// 3. Factory methods
public static Car createCar(String type) { ... }

// 4. Singleton pattern
public static Database getInstance() { ... }

Khi nào KHÔNG nên dùng static?

❌ KHÔNG NÊN DÙNG static trong các trường hợp sau: 
// 1. Trạng thái chia sẻ (dễ gây bug đa luồng)
public static List<String> users = new ArrayList<>();  // NGUY HIỂM!

// 2. Business logic (khó test)
public static void processOrder(Order order) { ... }  // KHÓ TEST!

// 3. Khi cần đa hình (không thể override)
class Animal {
    public static void makeSound() { ... }  // KHÔNG thể override
}

13.3 Object Class

Trong Java, mọi lớp đều kế thừa từ lớp Object. Lớp Object là gốc của tất cả các lớp.
Mọi class đều extends Object 
// Mặc định, mọi class đều extends Object
class MyClass { 
    // Tương đương với: class MyClass extends Object
}

1. toString() - Biểu diễn chuỗi

Mục đích: Trả về biểu diễn chuỗi của đối tượng.

Mặc định: getClass().getName() + "@" + Integer.toHexString(hashCode())

Student.java - So sánh có và không override toString() 
public class Student {
    private String id;
    private String name;
    private int age;
    
    public Student(String id, String name, int age) {
        this.id = id;
        this.name = name;
        this.age = age;
    }
    
    // ❌ KHÔNG override toString()
    public static void main(String[] args) {
        Student s = new Student("S001", "Nguyễn Văn A", 20);
        System.out.println(s);  // Student@1b6d3586 - VÔ NGHĨA!
    }
    
    // ✅ override toString()
    @Override
    public String toString() {
        return "Student{id='" + id + "', name='" + name + "', age=" + age + "}";
    }
    
    // Output: Student{id='S001', name='Nguyễn Văn A', age=20}
}

toString() được tự động gọi khi:

Student s = new Student("S001", "Nam", 20);

System.out.println(s);                    // Tự động gọi toString()
System.out.println("Student: " + s);      // Tự động gọi toString()
String info = "Info: " + s;              // Tự động gọi toString()

2. equals() - So sánh bằng nhau

Mục đích: So sánh hai đối tượng có bằng nhau không.

Vấn đề với toán tử == 
String s1 = new String("Hello");
String s2 = new String("Hello");

System.out.println(s1 == s2);      // false - so sánh địa chỉ
System.out.println(s1.equals(s2)); // true  - so sánh nội dung

Cách override equals() chuẩn:

Cách override equals() đúng chuẩn 
import java.util.Objects;

public class Student {
    private String studentId;
    private String name;
    private int age;
    
    @Override
    public boolean equals(Object obj) {
        // 1. Kiểm tra cùng tham chiếu
        if (this == obj) return true;
        
        // 2. Kiểm tra null
        if (obj == null) return false;
        
        // 3. Kiểm tra cùng lớp
        if (getClass() != obj.getClass()) return false;
        
        // 4. Ép kiểu
        Student other = (Student) obj;
        
        // 5. So sánh các trường quan trọng
        return Objects.equals(studentId, other.studentId);
    }
}

13.4 So sánh == và equals()

Hiểu bản chất: == so sánh địa chỉ | equals() so sánh nội dung (có thể override)

Bảng so sánh

Toán tử/Phương thức Primitive types Reference types
== So sánh giá trị So sánh địa chỉ bộ nhớ
equals() Không có So sánh nội dung (có thể override)

Ví dụ minh họa

Các trường hợp so sánh phổ biến 
// 1. Primitive types - chỉ có ==
int a = 5;
int b = 5;
System.out.println(a == b);  // true

// 2. Reference types
String s1 = new String("Hello");
String s2 = new String("Hello");
String s3 = s1;

System.out.println(s1 == s2);      // false - khác object
System.out.println(s1 == s3);      // true - cùng object
System.out.println(s1.equals(s2)); // true - cùng nội dung

// 3. String literal pool
String str1 = "Hello";
String str2 = "Hello";
System.out.println(str1 == str2);  // true! (cùng trong pool)

// 4. Wrapper classes
Integer i1 = 127;
Integer i2 = 127;
System.out.println(i1 == i2);      // true (cache -128 đến 127)

Integer i3 = 128;
Integer i4 = 128;
System.out.println(i3 == i4);      // false! (ngoài cache)
System.out.println(i3.equals(i4)); // true

Quy tắc vàng khi so sánh

Best practices khi so sánh object 
// 1. Luôn dùng equals() để so sánh nội dung object
if (obj1.equals(obj2)) { ... }

// 2. Cẩn thận với null
String s1 = null;
String s2 = "Hello";

// ❌ SAI: s1.equals(s2) - NullPointerException
// ✅ ĐÚNG: 
if (s1 != null && s1.equals(s2)) { ... }
// HOẶC:
import java.util.Objects;
if (Objects.equals(s1, s2)) { ... }

// 3. Với enum, có thể dùng == (an toàn, nhanh hơn)
enum Status { ACTIVE, INACTIVE }
Status s1 = Status.ACTIVE;
Status s2 = Status.ACTIVE;
System.out.println(s1 == s2);      // true (tốt với enum)
System.out.println(s1.equals(s2)); // true (cũng được)
